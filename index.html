Reference · 2024–2025

Modern CSS Features

A curated overview of the newest CSS capabilities — from layout primitives to logic functions.

⬜
Layout
Container Queries
+

Style elements based on their parent container's size — not just the viewport. A game-changer for reusable, truly modular components. Container units like cqw and cqi also let you size things relative to the container.

Example
.card {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 500px) {
  .card__body {
    display: flex;
    flex-direction: row;
    font-size: 1.2cqi; /* container query unit */
  }
}
🎯
Selectors
:has() — The Parent Selector
+

Select an element based on its children or descendants. Effectively CSS's first parent selector. Style a card differently if it has an image. Highlight a form if it contains invalid fields. Massive reduction in JavaScript for conditional styling.

Example
/* Card with an image gets a taller layout */
.card:has(img) {
  grid-template-rows: 200px 1fr;
}

/* Form with invalid fields turns red */
form:has(:invalid) {
  border: 2px solid crimson;
}

/* Nav item that contains the current page link */
li:has(a[aria-current="page"]) {
  font-weight: bold;
}
🪆
Scope & Layers
CSS Nesting
+

Write nested selectors natively in CSS without a preprocessor. Child rules inherit the parent selector context, making component styles far more readable and co-located.

Example
.card {
  background: white;
  padding: 1rem;

  &:hover {
    box-shadow: 0 4px 20px rgba(0,0,0,.1);
  }

  & h2 {
    font-size: 1.25rem;
  }

  @media (max-width: 600px) {
    padding: 0.5rem;
  }
}
🔒
Scope & Layers
@scope
+

Constrain styles to a specific subtree of the DOM. Prevents style leakage between components without needing BEM or CSS Modules. You can also define a "donut scope" — styles inside a root but outside a limit.

Example
@scope (.card) {
  h2 { color: teal; }
  p  { font-size: 0.9rem; }
}

/* Donut scope: inside .card but not inside .card__footer */
@scope (.card) to (.card__footer) {
  a { color: royalblue; }
}
🗂️
Scope & Layers
@layer — Cascade Layers
+

Explicitly control the cascade by grouping styles into named layers. Later-declared layers win regardless of specificity. Ideal for managing resets, third-party libraries, and component overrides predictably.

Example
/* Declare order — later = higher priority */
@layer reset, base, components, utilities;

@layer reset {
  * { margin: 0; box-sizing: border-box; }
}

@layer components {
  .btn { padding: .5rem 1rem; }
}

@layer utilities {
  /* wins over components, even with same specificity */
  .p-0 { padding: 0; }
}
⊞
Layout
Subgrid
+

Let a nested grid element participate in its parent grid's tracks. This means child elements can align to the grandparent grid's columns/rows — no more manual track duplication for complex card layouts.

Example
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto 1fr auto;
}

.card {
  display: grid;
  grid-row: span 3;
  /* inherit parent's row tracks */
  grid-template-rows: subgrid;
}
/* Now header/body/footer of each card align across columns */
✨
Animation
View Transitions API
+

Animate between two DOM states — or even between page navigations — with a smooth crossfade or custom transition. Previously required complex FLIP JavaScript techniques. Cross-document transitions are now arriving too.

Example
/* Trigger in JS */
document.startViewTransition(() => {
  updateTheDom();
});

/* Customize the transition in CSS */
::view-transition-old(root) {
  animation: fade-out 0.3s ease;
}
::view-transition-new(root) {
  animation: slide-in 0.3s ease;
}

/* Named transition for a specific element */
.hero-image {
  view-transition-name: hero;
}
🎞️
Animation
Scroll-Driven Animations
+

Tie CSS animation progress directly to scroll position or element visibility — all in CSS, no scroll event listeners. Use animation-timeline: scroll() for scroll position or view() for element entry/exit.

Example
/* Progress bar tied to page scroll */
.progress-bar {
  animation: grow linear;
  animation-timeline: scroll(root block);
}

/* Fade in as element enters the viewport */
.section {
  animation: fade-in linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
}

@keyframes grow {
  from { transform: scaleX(0); }
  to   { transform: scaleX(1); }
}
↕️
Animation
Animate to auto / interpolate-size
+

For the first time, you can transition height: 0 to height: auto natively in CSS. Use the interpolate-size property to opt-in globally, or use calc-size() for individual transitions.

Example
/* Global opt-in */
html {
  interpolate-size: allow-keywords;
}

.accordion-body {
  height: 0;
  overflow: hidden;
  transition: height 0.35s ease;
}
.accordion-body.open {
  height: auto; /* now animates! */
}

/* Alternative: calc-size() per element */
.item.open {
  height: calc-size(auto, size);
}
⚓
Layout
Anchor Positioning
+

Tether an absolutely-positioned element to another element (the anchor) anywhere in the DOM — regardless of stacking context. Perfect for tooltips, popovers, and dropdowns that need to track a trigger element.

Example
.trigger {
  anchor-name: --my-anchor;
}

.tooltip {
  position: absolute;
  position-anchor: --my-anchor;

  /* align tooltip bottom to trigger's top */
  bottom: anchor(top);
  left: anchor(center);
  transform: translateX(-50%);
}
🎨
Color
Relative Color Syntax
+

Derive a new color from an existing one by manipulating its individual channels — without a preprocessor. You can lighten, darken, desaturate, or change the alpha of any color using the from keyword.

Example
--primary: #3b82f6;

.btn {
  background: var(--primary);

  /* 30% lighter version */
  color: oklch(from var(--primary) calc(l + 0.3) c h);

  /* 50% transparent version */
  box-shadow: 0 0 0 4px oklch(from var(--primary) l c h / 50%);
}
🧪
Color
color-mix()
+

Mix two colors together in a specified color space with a defined ratio. Useful for creating tints, shades, and intermediate palette values directly in CSS.

Example
/* 30% white mixed with the primary → a tint */
--tint: color-mix(in oklch, var(--primary), white 30%);

/* Semi-transparent overlay */
--overlay: color-mix(in srgb, black 40%, transparent);

.badge {
  background: color-mix(in hsl, royalblue, coral 50%);
}
🌈
Color
New Color Spaces (oklch, lab, display-p3…)
+

CSS now supports wide-gamut color spaces beyond sRGB: oklch, oklab, lab, lch, display-p3, and more. oklch is perceptually uniform — adjusting lightness doesn't shift the perceived hue, making it ideal for design systems.

Example
/* oklch(lightness chroma hue) */
--brand: oklch(0.65 0.2 260);

/* Vibrant P3 color — impossible in sRGB */
--vivid-green: color(display-p3 0.1 0.9 0.3);

/* Fallback for non-supporting browsers */
.btn {
  background: #22c55e;         /* sRGB fallback */
  background: var(--vivid-green); /* P3 override */
}
🔀
Logic
if() — CSS Conditionals
+

The first explicit branching function in CSS. Returns different values depending on conditions — style queries, media queries, or supports queries. Reduces the need for multiple selector rules just to swap a single value.

Example
.btn {
  background: if(
    style(--variant: primary): royalblue;
    style(--variant: danger):  crimson;
    else:                       gray;
  );
}

/* Usage */
<button style="--variant: danger">Delete</button>
𝑓
Logic
@function — Custom CSS Functions
+

Define your own CSS value functions. Unlike custom properties, @function can accept arguments and perform calculations, returning a single value. Brings programming-style abstraction to CSS without a preprocessor.

Example
@function --fluid-size(--min, --max) {
  result: clamp(
    var(--min),
    calc(var(--min) + (var(--max) - var(--min)) * 0.5vw),
    var(--max)
  );
}

h1 {
  font-size: --fluid-size(1.5rem, 3rem);
}
p {
  font-size: --fluid-size(1rem, 1.25rem);
}
🧩
Selectors
:is() and :where()
+

Both accept a selector list and match any element in it. :is() adopts the specificity of its most specific argument; :where() always has zero specificity — great for resets and base styles you want easily overridden.

Example
/* Instead of h1,h2,h3,h4 { } */
:is(h1, h2, h3, h4) {
  line-height: 1.2;
  font-weight: 600;
}

/* Zero-specificity base styles — easily overridden */
:where(article, section, aside) p {
  margin-bottom: 1em;
}

/* Complex selectors simplified */
a:is(:hover, :focus) {
  color: royalblue;
}

Typography
text-wrap: balance & pretty
+

Control how text breaks across lines. balance distributes text evenly so no line is dramatically shorter (great for headings). pretty avoids orphaned single words on the last line of a paragraph.

Example
/* Balanced heading — no awkward short last line */
h1, h2, h3 {
  text-wrap: balance;
}

/* No orphan at end of paragraphs */
p {
  text-wrap: pretty;
}

Forms & UI
field-sizing: content
+

Make form inputs and textareas automatically grow to fit their content, without any JavaScript. Replaces the classic JS auto-resize textarea hack with a single CSS property.

Example
textarea {
  field-sizing: content;
  min-height: 3lh;  /* at least 3 line heights */
  max-height: 20lh; /* cap growth */
}

/* Also works on text inputs — sizes to content */
input[type="text"] {
  field-sizing: content;
  min-width: 10ch;
}

Forms & UI
Styleable <select>
+

Opt into a fully styleable select menu — including the dropdown picker itself, not just the button. For years the opened dropdown was OS-rendered and uncustomizable; now you can style it like any other element.

Example
/* Opt-in to base styles */
select, ::picker(select) {
  appearance: base-select;
}

/* Now style freely */
select {
  border: 2px solid #3b82f6;
  border-radius: 8px;
  padding: 0.5rem 1rem;
}

::picker(select) {
  background: #1e1e2e;
  border-radius: 10px;
  box-shadow: 0 8px 40px rgba(0,0,0,.4);
}

Forms & UI
Popover API
+

A native HTML/CSS popover system. Add popover to any element to give it open/close behaviour at the top layer, with light-dismiss and accessibility built in. Style the backdrop and entry/exit with CSS.

Example
/* HTML */
<button popovertarget="my-tip">Hover</button>
<div id="my-tip" popover>Tooltip content</div>

/* CSS */
[popover] {
  background: #222;
  border: none;
  border-radius: 8px;
  padding: 1rem;
}

[popover]::backdrop {
  background: rgba(0,0,0,0.4);
}

Layout
Dynamic Viewport Units (dvh, svh, lvh)
+

Mobile browsers shrink/grow their chrome (address bar) while scrolling, making plain 100vh unreliable. New units solve this: svh (smallest), lvh (largest), and dvh (dynamic — updates as the browser chrome resizes).

Example
/* Old approach — breaks on mobile */
.hero { height: 100vh; }

/* dvh: always matches visible viewport height */
.hero { height: 100dvh; }

/* svh: uses the smallest (browser chrome visible) */
/* lvh: uses the largest (browser chrome hidden) */
.full-screen {
  min-height: 100svh;
  max-height: 100lvh;
}
🔢
Selectors
sibling-count() & sibling-index()
+

Expose the total number of siblings and an element's own index as numeric values directly in CSS — no style="--index: N" hacks in HTML required. Great for staggered animations and auto-calculated layouts.

Example
li {
  /* Stagger animation delay by index */
  animation-delay: calc(sibling-index() * 0.1s);

  /* Hue shift across all items */
  color: hsl(
    calc(sibling-index() / sibling-count() * 360deg)
    70% 60%
  );
}
